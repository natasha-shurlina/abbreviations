# Проверка детектора

---

## det_testing.py

Берёт папку с текстовыми файлами `.txt`, прогоняет каждый через детектор
сокращений и сохраняет результат в подпапку `results/`.
Сравнивает результаты с эталонными ответами из подпапки `gt/`
и вычисляет доли (0...1): сколько правильно найдено (true positive), сколько не найдено (false negative), сколько найдено лишнего (false positive) .

---

## Создание папки

Рядом с файлами проекта необходимо создать папку `texts/` и положить туда
текстовые файлы `.txt` которые необходимо проверить:

```
checker/
├── det.py
├── det_testing.py
├── llm_interface.py
├── llm_huggingface.py
├── rules.yaml
└── texts/
    ├── doc1.txt
    ├── doc2.txt
    └── doc3.txt
```

Для проверки проверки работы детекстора необходимо
создать подпапку `gt/` внутри `texts/`. В ней нужно создать файл
с теми же именами что и входные.

Содержимое каждого файла — список фрагментов которые детектор
должен был найти как ошибки, по одному на строку.

```
texts/
├── doc1.txt
├── doc2.txt
└── gt/
    ├── doc1.txt
    └── doc2.txt
```
Пример `gt/doc1.txt`:

```
# ожидаемые ошибки в doc1.txt
Разр.
ГИБ
кол-во
```

## Запуск проверки и сравнения

### Windows

Запуск проверки:

```bash
python det_testin.py -i texts/ -r rules.yaml
```

Скрипт автоматически создаст подпапку `results/` и запишет туда
результат для каждого файла.

В терминале для каждого файла выведется:

```
Обрабатываю: doc1.txt
Ошибок найдено: 4
```


Запуск сравнения:


```bash
python det_testin.py -i texts/ -r rules.yaml --gt
```

В терминале для каждого файла выведется:

```
Обрабатываю: doc1.txt
ошибок найдено: 4, ожидалось: 4, ttrue_positive: 1.0, false_negative: 0.0, false_positive: 0.0
```

В каждый `.json` добавится блок `comparison`:

```json
"comparison": {
  "true_positive": 0.5,
  "false_negative": 0.167,
  "false_positive": 0.333
}
```

### macOS/Linux

Запуск проверки:

```bash
python3 det_testin.py -i texts/ -r rules.yaml
```

Запуск сравнения:


```bash
python3 det_testin.py -i texts/ -r rules.yaml --gt
```

---

## Метрики

`true_positive` - доля правильно найденных ошибок
`false_negative` - доля пропущенных ошибок 
`false_positive` - доля лишних срабатываний 

В сумме дают 1.0.


## Итоговая структура папок

```
checker/

    │   ├── doc1.txt
    │   └── doc2.txt
    └── results/

        └── doc2.json
```
