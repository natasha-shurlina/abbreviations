# Проверка сокращений в инженерных документах

Программа проверяет текстовые и markdown-файлы на соответствие сокращений
правилам ГОСТ 2.316 и ГОСТ 21.101. Результат — JSON-отчёт со списком ошибок
и корректных сокращений.

---

## Структура проект

det.py - Детерминированная проверка сокращений + вызов LLM

llm_interface.py - Абстрактный интерфейс LLM-провайдера

llm_huggingface.py - Реализация провайдера через Hugging Face Inference API

rules.yaml - Словарь сокращений: полное написание и допустимые варианты

sample.md - Пример входного файла для проверки

README.md - Документация

LIMITATIONS.md - Ограничения программы

examples.md - Пример вывода результата

---

## Алгоритм работы

**Детерминированный алгоритм** (`det.py`) находит кандидатов на сокращение
тремя регулярными выражениями: слова с точкой, аббревиатуры (2–3 буквы), 
короткие слова. Каждый кандидат проверяется по словарю `rules.yaml`:
сначала точное совпадение, затем составные сокращения через дефис и пробел,
затем: по префиксу без точек, потом расстоянием Левенштейна с порогом 2.

**LLM** (`llm_huggingface.py`) подключается опционально флагом `--llm`.
Для каждого сокращения с вердиктом ОШИБКА или НЕРАСПОЗНАНО вырезается
контекст ±100 символов и отправляется запрос к модели на Hugging Face.
Провайдер реализует абстрактный интерфейс `LLMProvider` из `llm_interface.py`,
что позволяет сменить модель без изменения основного кода.

## Требования

- Python 3.8+
- `pyyaml` — обязательно
- `huggingface_hub` — для LLM
- `requests` — для LLM (устанавливается как зависимость huggingface_hub)

---

## Установка и запуск

### Windows

**Установка зависимости:**
```cmd
pip install pyyaml huggingface_hub
```

**Запуск без LLM, вывод в терминал:**
```cmd
python det.py -i sample.md -r rules.yaml
```

**Запуск с LLM, вывод в терминал:**
```cmd
set HF_TOKEN=hf_ваш_токен
python det.py -i sample.md -r rules.yaml --llm
```

**Запуск без LLM, с сохранением отчёта:**
```cmd
python det.py -i sample.md -r rules.yaml -o result.json
```

**Запуск с LLM, с сохранением отчёта:**
```cmd
set HF_TOKEN=hf_ваш_токен
python det.py -i sample.md -r rules.yaml --llm -o result.json
```

## Входные файлы

**`sample.md`** — проверяемый текст в формате Markdown.
Markdown-разметка (заголовки, ссылки, таблицы) удаляется перед анализом,
позиции символов при этом сохраняются.

**`rules.yaml`** — словарь сокращений. Каждая запись содержит полное написание
и список допустимых сокращений:

```yaml
- full: "Разработал"
  allowed: ["разраб."]

- full: "Документ"
  allowed: ["докум.", "док."]
```

## Выходной JSON

```json
{
  "mode": "det",
  "input": "sample.md",
  "errors": [
    {
      "fragment": "Разр.",
      "position": 142,
      "expected": "разраб.",
      "verdict": "ОШИБКА",
      "source": "det",
      "note": "Возможно, имелось в виду: разраб. (полное слово: Разработал)",
      "llm_note": "неправильное сокращение, возможно сокращено слово разработал"
    }
  ],
  "skipped_count": 38,
  "allowed_found": ["Гл. инж.", "КПД", "разраб.", "пров."]
}
```

`errors` - Сокращения с вердиктом ОШИБКА
`skipped_count` - Количество слов, не похожих на сокращения
`allowed_found` - Список корректных сокращений, найденных в тексте
`llm_note` - Комментарий LLM
